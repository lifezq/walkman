**🎵 Walkman 音乐播放器 - 专注本地音乐播放**

Walkman 是一款专为音乐爱好者打造的本地音乐播放器，无需联网即可畅享高品质音乐体验。

**核心功能：**

✨ **本地音乐管理**
- 自动扫描设备本地音乐文件

📋 **歌单管理**
- 自定义创建多个歌单
- 快速添加歌曲到歌单
- 支持歌单内歌曲拖拽排序
- 最近播放历史记录

🎤 **歌词显示**
- 自动加载本地 LRC 歌词文件
- 支持配置在线歌词接口
- 歌词实时滚动显示
- 全屏播放页面沉浸式体验

❤️ **收藏与喜欢**
- 为喜爱的歌曲添加喜欢标记
- 收藏功能快速筛选常听歌曲
- 搜索功能快速定位歌曲

📤 **文件上传（可选）**
- 支持配置上传服务器地址
- 便于音乐文件同步备份

**应用特色：**
- 🚫 无广告、无内购、无会员
- 🎨 Material Design 设计风格
- 🔒 隐私安全，所有数据本地存储
- ⚡ 轻量流畅，占用资源少
- 📱 支持 Android 和 iOS 双平台

**适用人群：**
- 喜欢听本地音乐的用户
- 追求纯粹音乐体验的音乐爱好者
- 需要无网播放音乐的场景（地铁、飞机等）
- 拥有高品质音频文件的发烧友

让 Walkman 陪你享受纯粹的音乐时光！

## 权限说明

### Android 权限清单及说明

| 权限 | 用途说明 | 是否必须 |
|------|---------|---------|
| `WAKE_LOCK` | 保持屏幕唤醒，后台播放音乐 | 是 |
| `FOREGROUND_SERVICE` | 前台服务，后台播放音乐 | 是 |
| `FOREGROUND_SERVICE_MEDIA_PLAYBACK` | 媒体播放前台服务 | 是 |
| `READ_MEDIA_AUDIO` | 读取音频文件（Android 13+） | 是 |
| `READ_EXTERNAL_STORAGE` | 读取外部存储音乐文件（Android 12 及以下） | 是 |
| `POST_NOTIFICATIONS` | 显示播放通知 | 否（可选） |
| `INTERNET` | 在线歌词和文件上传功能 | 否（可选） |

## 技术准备清单

### 1. 应用签名

#### Android 签名
```bash
# 生成密钥库
keytool -genkey -v -keystore walkman-release.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias walkman

# 配置 android/key.properties
storePassword=你的密钥库密码
keyPassword=你的密钥密码
keyAlias=walkman
storeFile=../walkman-release.jks
```

在 `android/app/build.gradle.kts` 添加签名配置：
```kotlin
android {
    signingConfigs {
        create("release") {
            storeFile = file("../../walkman-release.jks")
            storePassword = System.getenv("KEYSTORE_PASSWORD")
            keyAlias = "walkman"
            keyPassword = System.getenv("KEY_PASSWORD")
        }
    }
    buildTypes {
        release {
            signingConfig = signingConfigs.getByName("release")
        }
    }
}
```

#### iOS 签名
- 需要 Apple Developer 账号
- 需要创建 App ID
- 需要创建证书和描述文件
- 在 Xcode 中配置 Signing & Capabilities

### 4. 混淆配置

#### Android ProGuard
创建 `android/app/proguard-rules.pro`：
```proguard
# Flutter 相关
-keep class io.flutter.** { *; }
-keep class androidx.** { *; }

# Just Audio
-keep class com.ryanheise.just_audio.** { *; }
-keep class com.google.android.exoplayer2.** { *; }

# Audio Service
-keep class com.ryanheise.audioservice.** { *; }
```

在 `build.gradle.kts` 启用混淆：
```kotlin
buildTypes {
    release {
        isMinifyEnabled = true
        isShrinkResources = true
        proguardFiles(
            getDefaultProguardFile("proguard-android-optimize.txt"),
            "proguard-rules.pro"
        )
    }
}
```

### 5. 版本号管理

修改 `pubspec.yaml`：

```yaml
version: 1.0.0+1
# 1.0.0 是版本名称（versionName）
# +1 是版本号（versionCode/buildNumber）
```

## ⚠️ 常见审核被拒原因

### Android 应用市场
1. **隐私政策问题**
   - 缺少隐私政策
   - 隐私政策链接无效
   - 隐私政策内容不完整

2. **权限问题**
   - 权限说明不清楚
   - 强制要求非必要权限
   - 权限使用与描述不符

3. **功能问题**
   - 应用功能不完整
   - 存在明显 bug
   - 崩溃率高

4. **内容问题**
   - 应用描述与实际不符
   - 截图使用虚假图片
   - 包含违规内容

### iOS App Store
1. **功能不完整**
   - Demo 版本
   - 功能缺失
   - 明显 bug

2. **元数据问题**
   - 截图与应用不符
   - 描述虚假宣传
   - 关键词堆砌

3. **隐私问题**
   - 未提供隐私政策
   - 隐私说明不完整
   - 强制收集数据

4. **违反准则**
   - 使用私有 API
   - 包含热更新
   - 诱导评价
